## 2. Step-by-Step Guide: Initiating the Angular Frontend Project

The foundation of our interactive e-commerce platform begins with setting up a robust Angular project. Angular is a powerful, opinionated framework for building single-page client applications. We will leverage its structure and tooling to create a maintainable and scalable frontend.

**Prerequisites:**

  * Node.js and npm (Node Package Manager) installed.
  * Angular CLI installed globally: `npm install -g @angular/cli`



**1\. Creating a New Angular Workspace:**

Open your terminal or command prompt and navigate to the directory where you want to create your project. Execute the following command to generate a new Angular workspace and initial application:
    
    
    ng new ecommerce-frontend --style=scss --routing=true

This command does the following:

  * `ng new ecommerce-frontend`: Creates a new Angular workspace named `ecommerce-frontend`.
  * `--style=scss`: Configures the project to use SCSS for styling, offering more advanced features than plain CSS.
  * `--routing=true`: Sets up the Angular Router module, which is essential for navigating between different views in our application (e.g., home page, product details, login page).



The CLI will prompt you with a few questions:

  * **Would you like to add Angular routing?** (Type `y` and press Enter)
  * **Which stylesheet format would you like to use?** (Select SCSS using the arrow keys and press Enter)



After answering, the Angular CLI will create the project structure and install all necessary dependencies. This process might take a few minutes.

**2\. Navigating to the Project Directory:**

Once the installation is complete, change your directory to the newly created project:
    
    
    cd ecommerce-frontend

**3\. Running the Development Server:**

To verify that the setup is successful, start the Angular development server:
    
    
    ng serve --open

This command compiles the application, starts a local development server, and automatically opens the application in your default web browser. You should see the default Angular welcome page.

**4\. Understanding the Project Structure:**

Familiarize yourself with the generated project structure:

  * `src/`: Contains all your application's source code.
  * `src/app/`: The root of your application components, modules, services, etc.
  * `src/app/app.component.html`: The main template for the root component.
  * `src/app/app.component.ts`: The root component's TypeScript file.
  * `src/app/app.module.ts`: The root application module, where you declare components and import other modules.
  * `src/app/app-routing.module.ts`: Configured by the CLI for routing.
  * `src/assets/`: For static assets like images and fonts.
  * `src/environments/`: Contains environment-specific configuration files.
  * `angular.json`: The workspace configuration file.
  * `package.json`: Lists project dependencies and scripts.



**5\. Initializing Git Repository (Optional but Recommended):**

If you haven't already, initialize a Git repository for version control:
    
    
    git init  
    git add .  
    git commit -m "Initial Angular project setup"

This initial setup provides a clean slate for building our e-commerce frontend. We have a functional development environment ready to host our components and services.

## 3. Designing and Implementing Product Listing Components

The product listing page is a cornerstone of any e-commerce platform, providing users with an overview of available items. In Angular, we achieve this by creating reusable components.

### **1\. Generating the Product List Component**

We'll create a dedicated component for displaying products. To generate the component, navigate to your project's root directory in the terminal and run:
    
    
    ng generate component components/product-list

This command creates a new folder `src/app/components/product-list` containing:

  * `product-list.component.html`: The template for displaying product information.

  * `product-list.component.ts`: The component's logic.

  * `product-list.component.scss`: The component's styles.

  * `product-list.component.spec.ts`: Unit tests for the component.




The component will also be automatically declared in `app.module.ts`.

### **2\. Defining the Product Interface**

Before we can display products, we need a clear structure for our product data. Create an interface to define the shape of a product object. In `src/app/models/product.interface.ts` (create the `models` folder if it doesn't exist):
    
    
    // src/app/models/product.interface.ts
    export interface Product {
        id: number;
        name: string;
        description: string;
        price: number;
        imageUrl: string;
        category: string;
    }

## 4. Crafting the Product Details View

When a user clicks on a product in the listing, they should be directed to a dedicated page with more comprehensive information. This requires creating a new component and configuring routing.

**1\. Generating the Product Detail Component:**

Create a component for the product details page:
    
    
    ng generate component components/product-detail

This will create `product-detail.component.html`, `.ts`, and `.scss` files.

**2\. Configuring Routing for Product Details:**

We need to define a route that accepts a product ID. Open `src/app/app-routing.module.ts` and update the routes array:
    
    
    // src/app/app-routing.module.ts
    import { NgModule } from '@angular/core';
    import { RouterModule, Routes } from '@angular/router';
    import { ProductListComponent } from './components/product-list/product-list.component';
    import { ProductDetailComponent } from './components/product-detail/product-detail.component';
    
    const routes: Routes = [
      { path: '', component: ProductListComponent }, // Default route
      { path: 'products/:id', component: ProductDetailComponent }, // Route with product ID parameter
      { path: '**', redirectTo: '' } // Wildcard route for unmatched paths
    ];
    
    @NgModule({
      imports: [RouterModule.forRoot(routes)],
      exports: [RouterModule]
    })
    export class AppRoutingModule { }

**3\. Implementing Product Detail Component Logic:**

The component needs to fetch the product ID from the URL and then retrieve the corresponding product data. We'll use the `ActivatedRoute` service for this.

In `product-detail.component.ts`:
    
    
    // src/app/components/product-detail/product-detail.component.ts
    import { Component, OnInit } from '@angular/core';
    import { ActivatedRoute } from '@angular/router';
    import { Product } from '../models/product.interface';
    import { PRODUCTS } from '../mock-data/products'; // Using mock data for now
    
    @Component({
      selector: 'app-product-detail',
      templateUrl: './product-detail.component.html',
      styleUrls: ['./product-detail.component.scss']
    })
    export class ProductDetailComponent implements OnInit {
      product: Product | undefined;
    
      constructor(private route: ActivatedRoute) { }
    
      ngOnInit(): void {
        // Get the product ID from the route parameters
        const productId = Number(this.route.snapshot.paramMap.get('id'));
    
        // Find the product in our mock data
        this.product = PRODUCTS.find(p => p.id === productId);
    
        // In a real application, you would call a service to fetch data from the API:
        // Example: this.productService.getProduct(productId).subscribe(data => this.product = data);
      }
    }

## 6. Implementing the Shopping Cart Functionality

The shopping cart is where users manage items they intend to purchase. This involves managing state on the client-side, potentially using a service to hold cart data.

### **1\. Creating a Cart Service**

A service is ideal for managing the shopping cart's state across different components.

To generate a new service, run the following command:
    
    
    ng generate service services/cart
    

In `src/app/services/cart.service.ts`:
    
    
    import { Injectable } from '@angular/core';
    import { BehaviorSubject, Observable } from 'rxjs';
    import { Product } from '../models/product.interface';
    
    interface CartItem {
      product: Product;
      quantity: number;
    }
    
    @Injectable({
      providedIn: 'root'
    })
    export class CartService {
      private cartItems = new BehaviorSubject<CartItem[]>([]);
      cartItems$: Observable<CartItem[]> = this.cartItems.asObservable();
    
      constructor() { }
    
      addToCart(product: Product): void {
        const currentCart = this.cartItems.getValue();
        const existingItemIndex = currentCart.findIndex(item => item.product.id === product.id);
    
        if (existingItemIndex > -1) {
          // If item exists, increment quantity
          currentCart[existingItemIndex].quantity++;
        } else {
          // Otherwise, add new item
          currentCart.push({ product: product, quantity: 1 });
        }
    
        this.cartItems.next([...currentCart]); // Emit new array to trigger updates
        console.log('Item added to cart:', product.name);
      }
    
      removeFromCart(productId: number): void {
        const currentCart = this.cartItems.getValue();
        const updatedCart = currentCart.filter(item => item.product.id !== productId);
        this.cartItems.next(updatedCart);
        console.log('Item removed from cart:', productId);
      }
    
      updateQuantity(productId: number, quantity: number): void {
        const currentCart = this.cartItems.getValue();
        const itemIndex = currentCart.findIndex(item => item.product.id === productId);
    
        if (itemIndex > -1) {
          if (quantity > 0) {
            currentCart[itemIndex].quantity = quantity;
            this.cartItems.next([...currentCart]);
          } else {
            this.removeFromCart(productId);
          }
        }
      }
    
      clearCart(): void {
        this.cartItems.next([]);
        console.log('Cart cleared.');
      }
    
      getCartTotal(): Observable<number> {
        return new Observable(observer => {
          this.cartItems$.subscribe(items => {
            const total = items.reduce((sum, item) => sum + (item.product.price * item.quantity), 0);
            observer.next(total);
          });
        });
      }
    }

## 8. Implementing Client-Side Authentication Flow with JWT

Securing our application requires implementing a client-side authentication flow that utilizes JSON Web Tokens (JWT) issued by the backend. This involves storing the token and attaching it to subsequent authenticated requests.

**1\. Authentication Service:**

Create a dedicated service to manage authentication state and token handling.

In `src/app/services/auth.service.ts`:
    
    
    import { Injectable } from '@angular/core';
    import { HttpClient, HttpHeaders } from '@angular/common/http';
    import { BehaviorSubject, Observable, tap } from 'rxjs';
    import { Router } from '@angular/router';
    
    @Injectable({
      providedIn: 'root'
    })
    export class AuthService {
      private apiUrl = 'https://localhost:7001/api'; // Your API URL
      private tokenKey = 'authToken';
      private currentUserSubject = new BehaviorSubject(null);
      currentUser$: Observable = this.currentUserSubject.asObservable();
    
      constructor(private http: HttpClient, private router: Router) { 
        // Check for existing token on initialization
        const token = localStorage.getItem(this.tokenKey);
        if (token) {
          // Optionally decode token to get user info and set currentUserSubject
          // For simplicity, we'll just assume a token means a logged-in user
          this.currentUserSubject.next({ token }); 
        }
      }
    
      register(user: any): Observable {
        return this.http.post(`${this.apiUrl}/auth/register`, user);
      }
    
      login(credentials: any): Observable<{ token: string, userId: number, username: string }> {
        return this.http.post<{ token: string, userId: number, username: string }>(`${this.apiUrl}/auth/login`, credentials).pipe(
          tap(response => {
            // Store token and user info
            localStorage.setItem(this.tokenKey, response.token);
            localStorage.setItem('userId', response.userId.toString());
            localStorage.setItem('username', response.username);
            this.currentUserSubject.next({ token: response.token, userId: response.userId, username: response.username });
            console.log('Login successful, token stored.');
          })
        );
      }
    
      logout(): void {
        localStorage.removeItem(this.tokenKey);
        localStorage.removeItem('userId');
        localStorage.removeItem('username');
        this.currentUserSubject.next(null);
        console.log('User logged out.');
        this.router.navigate(['/login']); // Redirect to login page
      }
    
      getToken(): string | null {
        return localStorage.getItem(this.tokenKey);
      }
    
      isAuthenticated(): boolean {
        return !!this.getToken();
      }
    
      getCurrentUser(): Observable<{ token: string, userId: number, username: string } | null> {
        return this.currentUser$;
      }
    
      // Method to add token to headers for authenticated requests
      getAuthHeaders(): HttpHeaders {
        const token = this.getToken();
        if (token) {
          return new HttpHeaders({
            'Authorization': `Bearer ${token}`
          });
        }
        return new HttpHeaders(); // Return empty headers if no token
      }
    }

    import { Injectable } from '@angular/core';
    import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot, UrlTree, Router } from '@angular/router';
    import { Observable } from 'rxjs';
    import { AuthService } from './services/auth.service';
    
    @Injectable({
      providedIn: 'root'
    })
    export class AuthGuard implements CanActivate {
      constructor(private authService: AuthService, private router: Router) {}
    
      canActivate(
        route: ActivatedRouteSnapshot,
        state: RouterStateSnapshot): Observable | Promise | boolean | UrlTree {
        
        if (this.authService.isAuthenticated()) {
          return true; // User is authenticated, allow access
        }
    
        // User is not authenticated, redirect to login page
        this.router.navigate(['/login']);
        return false;
      }
    }

  * **Console:** Check for JavaScript errors, log messages from your application (e.g., from `console.log` statements in services and components).
  * **Network Tab:** Monitor all HTTP requests made by the frontend to the backend. Inspect request payloads, headers (especially Authorization headers), and response statuses (e.g., 200 OK, 400 Bad Request, 401 Unauthorized, 500 Internal Server Error). This is vital for debugging API integrations.
  * **Application Tab (or Storage):** Inspect `localStorage` to verify that JWT tokens and user information are being stored correctly.
  * **Elements Tab:** Inspect the HTML structure and CSS styles to debug UI issues.



**3\. Debugging Techniques:**

  * **Breakpoints:** Set breakpoints in your TypeScript code within the browser's developer tools to step through execution line by line, inspect variable values, and understand the flow of logic.
  * **Console Logging:** Strategically place `console.log()` statements to track variable states and execution paths, especially in asynchronous operations.
  * **Angular DevTools (Browser Extension):** Install the Angular DevTools browser extension for Chrome/Firefox. It provides insights into component state, dependency injection, and routing, significantly aiding debugging.
  * **Backend Logs:** Correlate frontend issues with logs generated by your ASP.NET Core backend. If the Network tab shows a 500 error, check your backend logs for the specific exception.



**4\. Example Debugging Scenario: Cart Total Incorrect**

**Issue:** The cart total displayed on the frontend is incorrect.

**Debugging Steps:**

  1. **Frontend Cart Service:** Inspect the `getCartTotal()` method in `cart.service.ts`. Ensure the calculation logic is correct (price * quantity). Use `console.log` within the subscription to see the intermediate `items` array and the calculated `total`.
  2. **Network Tab:** Verify that the product prices being fetched from the API are correct. If prices are wrong here, the issue is likely in the backend data.
  3. **Cart Display Component:** Check how the `total$` observable is used in `cart-display.component.html`. Ensure the correct pipe (e.g., `number:'1.2-2'`) is applied.
  4. **Backend API:** If the issue persists, examine the API endpoint that provides product prices. Ensure it's returning accurate data.
